Duplicate Booking Detection Flow

The Race Condition

In high-demand flights, multiple users may attempt to book the same seat (e.g., 12A) simultaneously. A standard "Check-then-Act" database transaction is often too slow or complex to scale.

Solution: Use Redis Distributed Locks (Redlock pattern).

Atomic Operation: The SET NX command ensures only one request succeeds.

TTL Safety: The EX (Expiration) prevents deadlocks if a service crashes before releasing the lock.
