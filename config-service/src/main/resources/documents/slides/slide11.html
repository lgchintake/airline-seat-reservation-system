Resilience & Performance Patterns

Scalability Patterns

Handling increased load & throughput

Horizontal Pod Autoscaling (HPA)

Automatically scales microservice replicas based on CPU/Memory metrics in Kubernetes.

Kafka Partitioning

Partitions topics by FlightID to ensure parallel processing while maintaining ordering.

Redis Cluster

Sharded Redis instances for distributed caching and high-throughput locking.

Stateless Microservices

Services do not store session state, allowing any instance to handle any request.

Elasticsearch Indexing

Optimized inverted indices for sub-second flight search performance across millions of records.

Reliability Patterns

Fault tolerance & system stability

Circuit Breaker (Resilience4j)

Fails fast when downstream services (e.g., Payment Gateway) are unresponsive to prevent cascading failures.

Retry with Exponential Backoff

Automatically retries transient failures with increasing delays to avoid overwhelming recovering services.

Dead Letter Queue (DLQ)

Captures unprocessable Kafka messages for manual inspection, preventing consumer lag.

Health Checks & Probes

Kubernetes Liveness/Readiness probes ensure traffic only goes to healthy pods.

Distributed Tracing (Zipkin/OTel)

End-to-end request tracking to quickly identify performance bottlenecks and error roots.
