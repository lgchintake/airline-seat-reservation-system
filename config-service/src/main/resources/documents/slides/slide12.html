System Summary

Key Benefits & Achievements

Event-Driven Architecture

Loosely coupled microservices communicating via Kafka ensure system resilience and flexibility.

Robust Concurrency Control

Redis-based distributed seat locking mechanism effectively prevents overbooking scenarios.

Fault Tolerance

Async processing via Kafka allows the system to handle spikes and recover gracefully from failures.

Data Isolation

Database-per-service pattern ensures loose coupling and allows independent scaling of storage layers.

High Scalability

Stateless services and horizontal scaling capabilities support high booking traffic loads.

Quality Assurance

70%+ unit test coverage achieved using JUnit 5 and Mockito.
